


在Ubuntu 20.04本地环境中使用vLLM部署Qwen大模型时，虽然通过curl命令可以正常访问模型服务，但OpenClaw访问失败，**主要原因是OpenClaw的配置文件未正确设置vLLM服务地址或配置文件优先级问题**。以下是详细排查和解决方案：

## 一、根本原因分析

1. **配置文件优先级问题**  
   OpenClaw存在多层配置文件，优先级从高到低为：  
   - `models.json`（最高优先级，agent级别模型配置）  
   - `auth-profiles.json`（认证信息）  
   - `openclaw.json`（全局配置）  
   
   当用户只修改`openclaw.json`时，`models.json`中的旧配置仍然生效，会覆盖全局配置。

2. **vLLM服务地址配置错误**  
   OpenClaw默认配置可能指向错误的IP地址（如旧服务器IP），导致请求无法到达本地vLLM服务。

3. **安全机制触发**  
   OpenClaw内置了Cooldown冷却保护机制，当某个provider连续请求失败多次后，会临时屏蔽对该provider的请求，表现为"Connection error"。

## 二、解决方案

### 1. 检查并修改配置文件

**步骤一：确认vLLM服务正常运行**
```bash
# 检查端口监听情况
netstat -tuln | grep 8000

# 测试vLLM API（应返回模型列表）
curl http://127.0.0.1:8000/v1/models
```

**步骤二：检查OpenClaw配置文件**
```bash
# 查看当前使用的配置文件
openclaw models status
```

**步骤三：修改models.json（推荐方案）**
```bash
# 将旧IP替换为本地vLLM服务地址
sed -i 's/10.10.85.220/127.0.0.1/g' ~/.openclaw/agents/main/agent/models.json

# 确认修改成功
grep baseUrl ~/.openclaw/agents/main/agent/models.json
```

**步骤四：重启OpenClaw网关**
```bash
openclaw gateway restart
```

### 2. 完整配置示例

**修改`~/.openclaw/openclaw.json`文件**，确保包含以下关键配置：
```json
{
  "models": {
    "mode": "merge",
    "providers": {
      "vllm": {
        "baseUrl": "http://127.0.0.1:8000/v1",
        "apiKey": "sk-local",
        "api": "openai-completions",
        "models": [
          {
            "id": "qwen-7b",
            "name": "Qwen 7B",
            "contextWindow": 120000,
            "maxTokens": 8192
          }
        ]
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "vllm/qwen-7b"
      }
    }
  }
}
```

### 3. 验证配置是否生效

```bash
# 检查OpenClaw状态
openclaw status --all

# 测试与vLLM的连接
openclaw doctor

# 尝试通过OpenClaw调用模型
openclaw chat "hello"
```

## 三、常见问题及解决

1. **"Connection error"问题**  
   - **原因**：OpenClaw的Cooldown机制已触发  
   - **解决**：重启网关清除状态 `openclaw gateway restart`

2. **配置修改后不生效**  
   - **原因**：配置文件优先级问题  
   - **解决**：直接修改`models.json`而非`openclaw.json`

3. **安全警告问题**  
   - **原因**：OpenClaw默认启用安全机制  
   - **解决**：在配置中添加安全设置  
   ```json
   {
     "security": {
       "sandbox": {
         "mode": "all"
       }
     }
   }
   ```

4. **端口冲突问题**  
   - **原因**：18789端口被占用  
   - **解决**：修改OpenClaw端口  
   ```bash
   openclaw gateway --port 18790
   ```

## 四、安全使用建议

1. **环境隔离**：建议使用Docker或虚拟机运行OpenClaw，避免在日常办公电脑上直接安装。

2. **权限控制**：不要使用管理员权限运行OpenClaw，创建专用低权限账户。

3. **网络限制**：配置为仅本地访问（127.0.0.1），避免将默认端口（18789/19890）暴露到公网。

4. **定期更新**：及时更新OpenClaw到最新版本，安装官方安全补丁。

如果按照上述步骤操作后问题仍然存在，建议检查OpenClaw日志获取更详细的错误信息：
```bash
cat ~/.openclaw/logs/openclaw.log
```

**关键提示**：OpenClaw的配置系统较为复杂，修改配置后务必重启网关服务才能使配置生效。
